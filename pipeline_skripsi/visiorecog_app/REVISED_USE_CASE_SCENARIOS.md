### **Diagram 1: Sistem Pengembangan Model**

| Nama Use Case                                       | **Training Model**                                                                                                                                                                                                                                                                                                                                                        |
| :-------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **ID**                                              | UC-01                                                                                                                                                                                                                                                                                                                                                                     |
| **Aktor**                                           | Pengembang                                                                                                                                                                                                                                                                                                                                                                |
| **Deskripsi**                                       | Use case ini mendeskripsikan proses bagi Pengembang untuk melatih model klasifikasi (KNN dan SVM) menggunakan data _embedding_ yang telah diekstrak dari galeri gambar.                                                                                                                                                                                                   |
| **Kondisi Awal**                                    | 1. Data _embedding_ dari galeri gambar telah diekstrak dan disimpan. <br> 2. Skrip untuk pelatihan model telah tersedia.                                                                                                                                                                                                                                                  |
| **Skenario Normal (Alur Utama)**                    | 1. **Pengembang** menjalankan skrip pelatihan model. <br> 2. **Sistem** memuat data _embedding_ dan label yang sesuai. <br> 3. **Sistem** melatih model KNN dengan parameter default. <br> 4. **Sistem** melatih model SVM dengan parameter default. <br> 5. **Sistem** menyimpan kedua model yang telah dilatih ke dalam file `.pkl`.                                    |
| **Skenario Alternatif (Eksperimen Hiperparameter)** | **3a. Eksperimen KNN:** Pengembang mengubah parameter `n_neighbors` dalam skrip untuk mencari jumlah tetangga terdekat yang optimal sebelum menjalankan pelatihan. <br> **4a. Eksperimen SVM:** Pengembang mengubah parameter `kernel` (misalnya dari 'rbf' ke 'linear') atau `C` (parameter regularisasi) untuk menemukan konfigurasi model SVM dengan performa terbaik. |
| **Kondisi Akhir**                                   | File `knn_model.pkl` dan `svm_model.pkl` telah dibuat atau diperbarui dan siap digunakan oleh aplikasi utama.                                                                                                                                                                                                                                                             |

### **Diagram 2: Sistem Pengenalan Wajah**

| Nama Use Case                                 | **Melakukan Rekognisi Wajah**                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| :-------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **ID**                                        | UC-02                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| **Aktor**                                     | Pengguna                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| **Deskripsi**                                 | Use case ini mendeskripsikan proses dari saat Pengguna mengunggah gambar wajah hingga sistem memproses dan menyiapkan hasil analisis untuk ditampilkan.                                                                                                                                                                                                                                                                                                                                           |
| **Kondisi Awal**                              | 1. Pengguna telah membuka aplikasi VisioRecog di browser. <br> 2. Server backend dan frontend sedang berjalan.                                                                                                                                                                                                                                                                                                                                                                                    |
| **Skenario Normal (Alur Utama)**              | 1. **Pengguna** mengunggah gambar wajah. <br> 2. **Sistem** (Frontend) mengirimkan gambar ke backend. <br> 3. **Sistem** (Backend) menjalankan `FaceRecognitionPipeline`. <br> 4. **Sistem** (Backend) mengekstrak _embedding_ dari gambar. <br> 5. **Sistem** (Backend) menggunakan _embedding_ tersebut untuk mendapatkan prediksi dari model klasifikasi (KNN dan SVM). <br> 6. **Sistem** (Backend) mengembalikan hasil analisis lengkap (termasuk restorasi, IQA, dan prediksi) ke frontend. |
| **Skenario Alternatif (Pencocokan Langsung)** | **5a. Pencocokan Cosine Similarity:** Secara paralel dengan klasifikasi, sistem juga melakukan pencocokan langsung (template matching) dengan menghitung jarak kosinus antara _embedding_ gambar uji dengan semua _embedding_ di galeri untuk menemukan kandidat terdekat. Hasil ini juga disertakan dalam respons akhir.                                                                                                                                                                         |
| **Kondisi Akhir**                             | Sistem (Frontend) telah menerima data hasil analisis yang lengkap dari backend dan siap untuk menampilkannya kepada Pengguna.                                                                                                                                                                                                                                                                                                                                                                     |

| :------------------------------------------ | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **ID**                                      | UC-03                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| **Aktor**                                   | Pengguna                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| **Deskripsi**                               | Use case ini mendeskripsikan proses bagi Pengguna untuk melakukan evaluasi kuantitatif terhadap kinerja model klasifikasi dengan mengunggah file JSON berisi data uji.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| **Kondisi Awal**                            | 1. Pengguna berada di tab "Evaluasi" pada antarmuka aplikasi. <br> 2. Pengguna memiliki file JSON yang berisi sekumpulan _embedding_ wajah dan label asli (ground truth) sebagai data uji.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| **Skenario Normal (Alur Utama)**            | 1. **Pengguna** mengunggah file JSON data uji melalui antarmuka. <br> 2. **Sistem** (Frontend) mengirimkan file JSON ke backend. <br> 3. **Sistem** (Backend) mem-parsing file JSON untuk mendapatkan daftar _embedding_ dan label aslinya. <br> 4. Untuk setiap _embedding_ dalam data uji, **Sistem** (Backend) melakukan prediksi menggunakan model yang telah dimuat (misalnya, KNN). <br> 5. **Sistem** (Backend) membandingkan hasil prediksi dengan label asli untuk menghitung metrik evaluasi. <br> 6. **Sistem** (Backend) menghitung **Confusion Matrix, Accuracy, Recall,** dan **F1-Score**. <br> 7. **Sistem** (Backend) mengembalikan metrik-metrik tersebut ke frontend. <br> 8. **Sistem** (Frontend) menampilkan nilai metrik dan visualisasi Confusion Matrix kepada Pengguna. |
| **Skenario Alternatif (Alur Pengecualian)** | **2a. Format JSON tidak sesuai:** Jika struktur file JSON yang diunggah tidak valid, sistem akan menampilkan pesan error yang menjelaskan format yang benar.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| **Kondisi Akhir**                           | Pengguna dapat melihat metrik kinerja kuantitatif dari model klasifikasi berdasarkan data uji yang diberikan.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
