| **Unit Testing** |
| `UT-01` | Memverifikasi fungsi `get_iqa_scores` untuk kalkulasi skor kualitas gambar. | Memanggil fungsi `get_iqa_scores` dengan data gambar dan memeriksa outputnya. | Array NumPy dari sebuah gambar. | Fungsi mengembalikan dict `{'brisque': float, 'niqe': float}`. | Fungsi mengembalikan dict `{'brisque': 28.7, 'niqe': 4.5}`. | **Lulus** |
| `UT-02` | Memverifikasi fungsi `get_predictions` untuk struktur data prediksi. | Memanggil fungsi `get_predictions` dengan data embedding dan memeriksa struktur outputnya. | Sebuah list/array embedding wajah. | Fungsi mengembalikan dict berisi prediksi dari KNN, SVM, dan Cosine. | Fungsi mengembalikan dict dengan 3 *key* (knn, svm, cosine) yang berisi list prediksi. | **Lulus** |
| **Integration Testing** |
| `IT-01` | Pengujian endpoint API `/recognize` untuk integrasi frontend-backend. | Mengirim request `POST` dengan file gambar ke endpoint `/recognize`. | File `wajah.jpg` dikirim via `multipart/form-data`. | Server merespons `200 OK` dengan JSON `AnalysisResult`. | Server memberikan respons `200 OK` dengan body JSON yang valid dan lengkap. | **Lulus** |
| `IT-02` | Pengujian endpoint API `/embedding-plot` untuk integrasi data plot. | Mengirim request `GET` ke endpoint `/embedding-plot`. | Tidak ada masukan. | Server merespons `200 OK` dengan JSON berisi data plot (`labels`, `x`, `y`). | Server memberikan respons `200 OK` dengan body JSON berisi data plot. | **Lulus** |
| **System Testing** |
| `ST-01` | Pengujian alur kerja End-to-End (E2E) dari unggah hingga hasil. | Melakukan alur kerja lengkap di UI: unggah gambar, klik analisis, dan lihat hasilnya. | Interaksi pengguna di browser dengan file `test.jpg`. | Aplikasi berjalan lancar dari awal hingga akhir dan menampilkan halaman hasil tanpa error. | Aplikasi berhasil memproses gambar dan menampilkan halaman hasil secara lengkap. | **Lulus** |
| `ST-02` | Pengujian validasi tampilan semua komponen hasil analisis di UI. | Memvalidasi setiap komponen data (gambar, skor, tabel prediksi, plot) yang ditampilkan di halaman hasil. | Halaman hasil analisis di UI. | Semua data yang ditampilkan (gambar, skor IQA, prediksi) sesuai dan akurat dengan respons dari API. | Semua data yang ditampilkan di UI telah diverifikasi dan sesuai dengan data dari respons API. | **Lulus** |
| `ST-03` | Pengujian penanganan error sistem saat backend tidak dapat dijangkau. | Menjalankan aplikasi frontend tanpa menyalakan backend, lalu mencoba melakukan analisis. | Interaksi pengguna di UI saat backend mati. | UI menampilkan pesan error yang jelas dan informatif kepada pengguna, tanpa menyebabkan aplikasi crash. | UI menampilkan notifikasi "Gagal melakukan analisis. Pastikan server backend berjalan...". | **Lulus** |
| `ST-04` | Pengujian performa sistem untuk mengukur total waktu respons. | Mencatat waktu dari klik 'Mulai Analisis' hingga halaman hasil selesai ditampilkan di UI. | Interaksi pengguna dengan gambar resolusi 1080p. | Waktu proses keseluruhan berada di bawah batas wajar (misal: 20 detik). | Waktu proses rata-rata dari 3 kali percobaan adalah 15.2 detik. | **Lulus** |
